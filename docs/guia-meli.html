Seção 01
Visão Geral do Sistema

O NEX Inventory é um sistema web de gestão de inventário para a operação logística de NODOs (agências parceiras) da Mercado Livre. Roda diretamente no navegador — sem instalação, sem app store — e funciona em celular, tablet e computador.

🏪
Agências (NODOs)
Pontos de coleta parceiros. Fazem inventário físico de pacotes e sacas, e expedem para o SVC.
🏭
Hub SVC
Centro de distribuição. Recebe de todas as agências, valida volumes e registra saídas.
👔
Time MELI
Administração central: cadastra NODOs, monitora operação e resolve divergências.

🎯 Problema que o NEX resolve

Antes do NEX, o controle de inventário era feito em planilhas e WhatsApp, sem rastreabilidade em tempo real e com alto risco de divergência de volumes entre agências e SVC.

Com o NEX, cada pacote é registrado digitalmente no momento da chegada ou expedição, com timestamp, operador e geolocalização — criando um trail auditável de toda a operação.

🌍

URL de acesso

luishenrique785.github.io/NEX — Salve nos favoritos do celular. Funciona como PWA (pode ser "instalado" na tela inicial do smartphone).

Seção 03
Fluxo Operacional Completo

A operação NEX segue um ciclo diário com três atores principais: Agência → SVC → MELI.

1

Agência recebe pacotes (Inventário Físico)

A agência escaneia cada pacote que chega. O sistema bloqueia duplicatas e mostra pendências do dia anterior.

2

Agência faz o Inventário de Sacas

Registra a chegada ou expedição de sacas com quantidade. Expedição exige placa + transportadora.

3

Agência expede para o SVC

Registra a expedição: escaneia cada pacote + informa motorista (CPF, nome, placa, transportadora). Pacotes saem do inventário ativo.

4

SVC recebe os pacotes

O operador SVC escaneia os pacotes que chegam. Confere totais com o manifesto físico antes de liberar o veículo.

5

SVC registra expedições

Ao enviar pacotes para o destino final, registra com placa e transportadora. Ciclo completo do pacote fica no banco.

6

MELI monitora e audita

Acessa o Supabase Dashboard para ver todos os registros, histórico completo e identificar divergências entre agências e SVC.

✅

Rastreabilidade em tempo real

Cada operação salva: timestamp, operador, NODO de origem, código do pacote. O Supabase mantém histórico completo com Row Level Security habilitado.

📍

Seleção de NODO por geolocalização

Ao entrar como "Agência", o app solicita permissão de localização. NODOs com coordenadas cadastradas dentro de 1 km aparecem automaticamente. Se nenhum for encontrado no raio, o operador pode buscar manualmente.

🚫

Anti-duplicata automático

O sistema impede que o mesmo código de pacote seja registrado duas vezes no inventário físico. Exibe alerta visual e sonoro ao detectar tentativa de duplicata.

Seção 05
Módulo SVC

O SVC é o hub central que recebe de todas as agências. Não seleciona NODO — opera globalmente sobre toda a rede.

OperaçãoDados obrigatóriosResultado no banco
Receber pacotes Código do pacote (scan/manual/foto) Registro em package_inventory com status "recebido_svc"
Expedir pacotes Código + Placa + Transportadora Atualização para status "expedido_svc" com metadados
Sacas — Chegada Quantidade Registro em bag_inventory tipo "chegada"
Sacas — Expedição Quantidade + Placa + Transportadora Registro em bag_inventory tipo "expedicao"

Seção 06
Gestão de NODOs

NODOs são as agências parceiras. Cada NODO tem um código único (ex: BRNMG838), nome, endereço e coordenadas geográficas para o filtro de proximidade.

🔒

Acesso restrito por PIN

A tela de gerenciamento de NODOs é protegida por PIN de administrador (configurado em src/config.ts). Apenas o time MELI deve ter acesso.

1

Exportar CSV da planilha

Na planilha Google Sheets (aba "BASE - Nodos"): Arquivo → Fazer download → CSV.

2

Acessar painel admin

Na tela inicial, toque em "NOVOS NODOS" e insira o PIN de administrador.

3

Upload do CSV

Selecione o CSV exportado. O sistema parseia automaticamente as colunas corretas da planilha.

4

Geocodificação automática

Para cada NODO com endereço, o sistema consulta o Nominatim (OpenStreetMap) para obter lat/lng. Rate limit: 1 req/segundo. Um lote de 50 NODOs leva ~1 minuto.

📊

Formato do CSV da planilha

A planilha "Controle NODOS - BASE - Nodos" tem este formato de colunas:

  • Coluna A: ETA (horário de chegada — ignorado)
  • Coluna B: Nó origem (sempre "NEX" — ignorado)
  • Coluna C: Facility NEx = Código do NODO (ex: BRNMG838)
  • Coluna D: Modal Veículo — ignorado
  • Coluna E: Nome Place = Nome do NODO
  • Coluna F: Endereço = Endereço completo para geocodificação

Seção 07
Banco de Dados — Estrutura

O banco está no Supabase (PostgreSQL). Todas as tabelas têm Row Level Security (RLS) habilitado com políticas de leitura/escrita via chave anônima.

TabelaDescriçãoColunas principais
nodos Cadastro de agências parceiras id, codigo, nome, endereco, cidade, estado, lat, lng
bag_inventory Movimentação de sacas id, nodo_id, tipo (chegada/expedicao), quantidade, placa, transportadora, created_at
package_inventory Inventário físico de pacotes por agência id, nodo_id, codigo_pacote, foto_url, status, created_at
package_expeditions Expedições de pacotes da agência para SVC id, nodo_id, codigo_pacote, motorista_nome, motorista_cpf, placa, transportadora, created_at
svc_receipts Recebimentos no SVC id, codigo_pacote, operador, created_at
svc_expeditions Expedições do SVC para destino final id, codigo_pacote, placa, transportadora, created_at
📸

Storage de fotos

Fotos de pacotes ficam no Supabase Storage no bucket "package-photos". URLs públicas são salvas na coluna foto_url da tabela package_inventory.

🔧

Como acessar o banco

Acesse supabase.com → Project: gjjwcpdkdqvfgephwulk → Table Editor para ver e filtrar todos os registros. Use o SQL Editor para queries avançadas e relatórios.

Seção 08
Deploy e Infraestrutura

O sistema é implantado automaticamente via GitHub Actions a cada push na branch main. Zero configuração de servidor necessária.

1

Push na branch main

Qualquer commit na branch main dispara o workflow automaticamente.

2

GitHub Actions executa o build

Instala dependências e roda npx expo export --platform web. Gera a pasta dist/ com HTML/JS/CSS estático.

3

Deploy automático para gh-pages

peaceiris/actions-gh-pages@v4 publica o conteúdo de dist/ na branch gh-pages.

4

Site disponível em ~2 minutos

GitHub Pages serve a branch gh-pages em luishenrique785.github.io/NEX. Usuários veem a nova versão ao atualizar a página.

⚠️

Configurações críticas do app.json

  • web.output: "single" — SPA mode, evita SSR que quebra no browser
  • experiments.baseUrl: "/NEX" — Necessário para assets corretos no GitHub Pages

🔑 Como fazer uma atualização

Para publicar qualquer mudança no sistema:

1. Edite o código no GitHub (UI web ou clone local)
2. Commit na branch main
3. Aguarde ~2 minutos
4. Verifique em luishenrique785.github.io/NEX

Não é necessário PC, npm ou qualquer ferramenta local — tudo roda via GitHub Actions na nuvem.

Seção 09
Importação de NODOs via CSV

A importação via Google Sheets direto pode falhar por restrições de CORS no browser. O método recomendado é fazer download do CSV e fazer upload no app.

1

Abra a planilha Google Sheets

Planilha: Controle NODOS → Aba: BASE - Nodos

2

Exporte como CSV

Arquivo → Fazer download → Valores separados por vírgula (.csv). Salve no celular ou computador.

3

Abra o painel admin no NEX

Na tela inicial → NOVOS NODOS → Digite o PIN de administrador.

4

Upload do CSV

Clique em "Selecionar arquivo CSV" → Selecione o arquivo baixado. O sistema parseia e inicia a importação automaticamente.

5

Aguarde geocodificação

Para cada NODO com endereço, o sistema consulta o Nominatim com throttle de 1s entre requests. Acompanhe o progresso na tela.

♻️

Idempotente — seguro para re-importar

O sistema verifica se o código do NODO já existe antes de inserir. Re-importar o mesmo CSV só adiciona os NODOs novos, sem duplicar os existentes.

Segurança

  • A chave anônima do Supabase (anon key) é pública por design — exposta no frontend. Isso é normal e seguro.
  • As políticas de Row Level Security (RLS) garantem que dados só são acessíveis com a chave correta.
  • O PIN de admin é verificado client-side — não é uma medida de segurança crítica, apenas um bloqueio operacional.
  • Para segurança real, considere adicionar autenticação Supabase Auth em versões futuras.
